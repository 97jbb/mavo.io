Documentation for HTML authors

Getting started

To use wysie to make a page editable, two things are needed as a minimum:

  • A data-store attribute with a URI that tells wysie where to store the data.
  • property attributes on every element that needs to be editable.

You can find details about these attributes below.

HTML attribute reference

data-store

A URL that tells wysie where to find the data. It’s exact format depends on the backend used. Each backend might also require more data-store-* properties.

You can use any URL you want, but if there is no supported backend associated with such URLs, the data will not be editable.

If the value is a hash (e.g. data-store="#todo"), wysie only stores the data locally. If the value is none, the data is not stored anywhere. This can be useful for experimenting, or for applications that don't require storage (e.g. a colour picker).

property or itemprop

The presence of any of these attributes on an element tells wysie that the element contains data and should be editable. The attribute value can be anything you want, as long as it’s different from other unrelated elements. It should describe the type of data the element will hold, because it will be used in various places in the editing interface. The rules for a good property name are the same as those for a good CSS class name.

data-multiple

Tells wysie that this element can be multiplied (i.e. is a collection). This way wysie knows to add controls for adding new elements, deleting existing ones and dragging to rearrange.

Some elements get this attribute automatically, which can be disabled with data-multiple="false". These elements are:

  • <li> without any other <li>s adjacent to it
  • <tr> without any other <tr>s adjacent to it

data-input

A CSS selector that points to a form element or HTML structure used for editing the current property. For more details on editing widgets see the section.

data-attribute

Overrides the default for which attribute (if any) contains the element’s data. Specify "null" for the data to be in the content.
