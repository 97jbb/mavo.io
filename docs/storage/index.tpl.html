Storage

The data-store attribute tells Mavo where any saved data will be stored.

Naming your application

It’s strongly recommended to name your application, for many reasons, such as storage, referring to properties in your Mavo from another Mavo, and many more. If you don't name your application, it will automatically get an id based on its index in the page, such as mavo1, mavo2 etc. As you can imagine, this is flimsy: You don't want to lose your data if you just rearrange your Mavos in the page or add more Mavo-enabled pages in the same website! You can name your application via any of the following attributes. If more than one of the following are present, this is also the order they are checked in.

  • data-mavo
  • property
  • itemprop
  • id
  • class (Only the first class is used)

Disabling storage

If your application does not need to store any data, you can use data-store="none", or the data-mavo attribute with no data-store attribute.

Local storage

You can save your data in the browser’s localStorage, by using the keyword local as the value of the data-store attribute (data-store="local").

This can be useful if you are not planning to share your data with other people. Please note that most localStorage implementations have a limit of 4MB. Also, localStorage is bound by the same-origin policy. This means that if you change your URL, your data will be left behind!
