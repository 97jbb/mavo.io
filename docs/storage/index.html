Storage

The data-store attribute tells Mavo where any saved data will be stored.

Naming your application

It‚Äôs strongly recommended to name your application, for many reasons, such as storage, referring to properties in your Mavo from another Mavo, and many more. If you don't name your application, it will automatically get an id based on its index in the page, such as mavo1, mavo2 etc. As you can imagine, this is flimsy: You don't want to lose your data if you just rearrange your Mavos in the page or add more Mavo-enabled pages in the same website! You can name your application via any of the following attributes. If more than one of the following are present, this is also the order they are checked in.

  • data-mavo
  • property
  • itemprop
  • id
  • class (Only the first class is used)

Disabling storage

If your application does not need to store any data, you can use data-store="none", or the data-mavo attribute with no data-store attribute.

Local storage

You can save your data in the browser‚Äôs localStorage, by using the keyword local as the value of the data-store attribute (data-store="local").

This can be useful if you are not planning to share your data with other people. Please note that most localStorage implementations have a limit of 4MB. Also, localStorage is bound by the same-origin policy. This means that if you change your URL, your data will be left behind!

Github

Mavo will provide login controls and will not allow any edits until the viewer has logged in and has commit permissions. If the file and/or repo does not exist, Mavo will create it upon saving. The value of the `data-store` attribute can take almost any Github file URL, as well as some more fuzzy ones:

https://github.com/[username]
This is perfect for people who want to use Github as a storage backend but are not familiar with it, since all it requires is signing up to get a username. Mavo will create a default repository of mv-data and will store data in a file called [mavo-id].json
https://github.com/[username]/[reponame]
Mavo will create a file called [mavo id].json in that repository.
https://github.com/[username]/[reponame]/[filename]
Mavo will assume a default branch of master
https://github.com/[username]/[reponame]/blob/[branch]/[filename]
The URL Github gives you when using its file browser. As specific as it gets, Mavo makes no assumptions.
https://[username].github.io/[reponame]/[filename]
Github Pages URL. Try to avoid these, as Mavo has to make some assumptions about what is the repo name, branch name, and filename, which could be wrong.
https://raw.githubusercontent.com/[username]/[reponame]/[branch]/[filename]
The URL that shows when you click the button Raw from the Github file browser.

Note that [filename] above could include a path, e.g. foo/bar/baz.json.
