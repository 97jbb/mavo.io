Storage

The mv-storage attribute tells Mavo where any saved data will be stored.

Disabling storage

To disable storage, just don't include an mv-storage attribute, or specify mv-storage="none". You could also disable storage on a per-property basis, by specifying mv-storage="none" on individual properties. This is useful when you don't need a property's value to be saved, e.g. for properties used for filtering data.

Local storage

You can save your data in the browser‚Äôs localStorage, by using the keyword local as the value of the mv-storage attribute (mv-storage="local").

This can be useful if you are not planning to share your data with other people. Please note that most localStorage implementations have a limit of 4MB. Also, localStorage is bound by the same-origin policy. This means that if you change your URL, your data will be left behind!

Github

Mavo will provide login controls and will not allow any edits until the viewer has logged in and has commit permissions. If the file and/or repo does not exist, Mavo will create it upon saving. The value of the `mv-storage` attribute can take almost any Github file URL, as well as some more fuzzy ones:

https://github.com/[username]
This is perfect for people who want to use Github as a storage backend but are not familiar with it, since all it requires is signing up to get a username. Mavo will create a default repository of mv-data and will store data in a file called [mavo-id].json
https://github.com/[username]/[reponame]
Mavo will create a file called [mavo id].json in that repository.
https://github.com/[username]/[reponame]/[filename]
Mavo will assume a default branch of master
https://github.com/[username]/[reponame]/blob/[branch]/[filename]
The URL Github gives you when using its file browser. As specific as it gets, Mavo makes no assumptions.
https://[username].github.io/[reponame]/[filename]
Github Pages URL. Try to avoid these, as Mavo has to make some assumptions about what is the repo name, branch name, and filename, which could be wrong.
https://raw.githubusercontent.com/[username]/[reponame]/[branch]/[filename]
The URL that shows when you click the button Raw from the Github file browser.

Note that [filename] above could include a path, e.g. foo/bar/baz.json.
