Defining a wysie app

The data-store attribute has two purposes: its presence tells wysie that it should pay attention to what is inside that element and its value tells wysie where to store any data. The value of the data-store attribute is a valid URL or a keyword and has the following meaning:

No value or "" or none
Disable storage.
local
Save in the browser (in localStorage).
#hash
A local hash can mean two things, depending on whether an element exists in the page with that id:
  1. If an element exists, that element will receive any saved data.
  2. If there is no corresponding element, the data will be saved in the browser (localStorage).
https://www.dropbox.com/s/[lettersandnumbers]/[filename]?dl=0
This is the URL that Dropbox gives you if you click “Share” on a file. In that case, the data will be saved back to Dropbox, and wysie will ask you to log in to edit or save.

The property attribute

The property attribute tells wysie which elements should be editable. Its value should describe the element, similarly to a id or class attribute. It can contain only letters, numbers and underscores (_).

Grouping

Sometimes the value of a property is not a simple piece of data that can be described with one element, but a group of elements. If an element with a property attribute contains other properties inside it, it becomes a group.

On their own, groups don’t do much, but they can be very useful when combined with lists and expressions, explained next.

Properties whose values are expressions are called “computed properties” and are not saved.

If there are groups and there are properties with the same name inside and outside the group, the value closest to your expression will be used, with parent groups having priority over inner groups.

Expression functions

References and simple arithmetic are useful, but not always enough. For this purpose, wysie defines a number of common math functions you can use inside expressions, such as: pow(base, exponent), round(number, decimals), sqrt(number) and many more.

Note that parentheses can be omitted if the entire expression is inside a function, to avoid awkwardness like =(pow(x, y))

You can use the iff(condition, ifyes, ifno) function to print out different text depending on a condition.

If what you’re printing contains characters other than letters, numbers and underscores, you will need to use quotes around it, just like in CSS font-family or HTML attributes.

There are functions that calculate a value out of a list of values (usually numbers):

Expression Meaning
count(property) Count how many items in the list actually have a value.
sum(property) Sum all numbers in the list.
average(property) Calculate the average of all numbers in the list.
min(property) Calculate the minimum of all numbers in the list.
max(property) Calculate the maximum of all numbers in the list.

Try adding and removing elements!

You can also do math with lists. For example, =sum(add(age, 5)) will add 5 to every age and then sum the result. There are many such functions that work well with lists (but not just lists) and are more human readable than their operator counterparts:

Math Function What it does
a + b add(a, b) Addition
a - b subtract(a, b) Subtraction
a * b multiply(a, b) Multiplication
a / b div(a, b) Division. Note that division by 0 will show up as ±∞ or NaN if it’s 0/0.
a && b and(a, b) Check that both a and b have a value
a || b or(a, b) Check that at least a or b have a value
!a not(a) Check if a doesn’t have a value
a < b lt(a, b) Check if a is less than b
a > b gt(a, b) Check if a is greater than b
a == b eq(a, b) Check if a is equal to b

If you have more operands you don’t need to nest these functions, just provide more parameters. E.g. you don’t need to do add(a, add(b, c)), add(a, b, c) will just work.

Finding mistakes

You can add a debug class on any element and after reloading you will get a table on every group inside it with all its expressions. The expressions in the table are editable and you can see the results of your edits as you type, both in the table and in your app. Hovering over the elements in the last column will highlight them in the app.

Summary/Cheatsheet

data-store attribute
Wysify an element and its contents. Its value specifies where data is stored.
property attribute
Name an element and make it editable, or group other properties.
data-multiple attribute
This element will be multiplied (i.e. create a list). Can only be used on elements with property or data-store attributes.
=foo
Print out the value of the "foo" property.
=(foo * sqrt(foo))
Print out the value of an expression. Can be used anywhere. Yes, in attributes too.
=count(foo), =sum(foo), =average(foo), =min(foo), =max(foo)
Display the count, sum, average, minimum or maximum of the different values of the "foo" property respectively. The "foo" property must be either a list (i.e. an element with data-multiple) or contained in a list for this to work.
class="debug"
Display an editable table with expressions and their current values in every group inside the class="debug" element.
