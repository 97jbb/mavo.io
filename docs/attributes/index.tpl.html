Attributes & Classes

This is an index of all Mavo attributes and classes, with links to their respective pages in the documentation, if any.

Core attributes

These attributes are crucial, most Mavo apps need them and elements with them are special.

Attribute Purpose Read more
data-mavo Allows you to assign an id to your app, which is used in a number of places. The element with this attribute is called the Mavo wrapper. Storage
data-store Tells Mavo where data is stored. The element with this attribute is called the Mavo wrapper. Storage
property Names elements that will be saved, edited, or used in expressions. Elements with this attribute are called Properties. Primer
typeof The value of this property element is not an attribute or its text content, but the union of the values of the properties inside it. This is almost always inferred from the structure of your properties and added automatically, but you can use it to force an element to be a group, or to declare the type of the group (e.g. typeof="Person") for better semantic value. Elements with this attribute are called groups. Primer
data-multiple Makes elements repeatable. Only has meaning on property elements. Elements with this attribute are called Collections. Primer

Other attributes

These are attributes you can use to further customize your Mavo app.

Attribute On Purpose Read more
data-attribute Properties and elements with data-content Customize which attribute data goes into on this element. For the element content, specify "null".
data-source Mavo wrapper If no data is stored, data-source will be used to load initial data. Same values as data-store. Storage
data-default Properties Sets the default value. If no value, the default value becomes the property content.
data-order Collections Values: desc or asc. Controls the order that new items are added to collections.
data-expressions Change the expression syntax or disable expressions altogether. MavoScript
data-content Set an element's content to the result of an expression.
data-edit Properties Customize how an element is edited by linking to another element, anywhere in the page.
data-input-* Properties Any attributes set that way will be copied over to the property's editor.

Author classes

In addition to attributes, you can also use these classes to customize the behavior of your Mavo app.

Name On Purpose Read more
autoedit Mavo wrapper Mavo will start in edit mode (no need to click the Edit button). Useful for apps that are almost always in edit mode, such as a to-do list.
debug Mavo wrapper or any group

Generated attributes & classes

You can use these attributes and classes in your CSS to customize your Mavo app. Classes are listed with a dot (.) before their name.

Name On Purpose Read more
.mv-wrapper Mavo wrapper
data-mv-progress Mavo wrapper Only set when an event is in progress. Its value reflects what event, e.g. "Saving" or "Loading".
data-clicked Properties that are <button> elements The number of times the button has been clicked.
.mv-ui Any (usually generated) controls for triggering Mavo-related actions
.mv-item Collection items Deleted but not saved items have the class .deleted.
.mv-item-controls
.mv-highlight
.mv-bar
.mv-editor Element used for editing a property
.mv-popup Generated popup for editing an attribute.
